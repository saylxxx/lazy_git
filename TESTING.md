**用途**: 快速檢查所有功能的基本狀態

### 2. unit-tests.sh - 單元測試
**用途**: 詳細測試各個功能的核心邏輯  
**執行時間**: 約 1-3 分鐘  
**適用場景**: 功能修改後的詳細驗證

```bash
# 執行所有單元測試
./unit-tests.sh

# 執行特定測試
./unit-tests.sh clean-lock
./unit-tests.sh clean-ab  
./unit-tests.sh branch-naming
./unit-tests.sh config
./unit-tests.sh update-ab
./unit-tests.sh file-structure
```

**測試項目**:
- 參數解析邏輯
- 分支命名規則
- 配置讀取功能
- 檔案操作邏輯

### 3. test-suite.sh - 完整測試套件
**用途**: 模擬真實環境的完整功能測試  
**執行時間**: 約 3-10 分鐘  
**適用場景**: 發布前的全面驗證

```bash
# 執行完整測試
./test-suite.sh

# 快速模式（跳過耗時測試）
./test-suite.sh --quick

# 詳細輸出模式
./test-suite.sh --verbose

# 只設置測試環境（不執行測試）
./test-suite.sh --setup-only

# 只清理測試環境
./test-suite.sh --cleanup-only
```

**測試項目**:
- 創建真實的 Git 倉庫環境
- 測試所有分支創建功能
- 測試所有清理功能
- 測試安裝流程
- 測試參數處理

## 使用建議

### 開發流程
1. **修改代碼後**: 執行 `./quick-verify.sh`
2. **功能完成後**: 執行 `./unit-tests.sh`
3. **發布前**: 執行 `./test-suite.sh`

### 除錯流程
如果測試失敗：

1. **查看詳細輸出**:
   ```bash
   ./test-suite.sh --verbose
   ```

2. **測試特定功能**:
   ```bash
   ./unit-tests.sh [功能名稱]
   ```

3. **手動測試環境**:
   ```bash
   ./test-suite.sh --setup-only
   # 手動在測試環境中除錯
   ./test-suite.sh --cleanup-only
   ```

## 常見問題

### Q: 測試失敗怎麼辦？
A: 
1. 檢查是否有足夠的磁碟空間
2. 確認 Git 已正確安裝
3. 檢查檔案權限
4. 使用 `--verbose` 模式查看詳細錯誤

### Q: 測試環境會影響系統嗎？
A: 不會。所有測試都在 `/tmp` 目錄下的臨時資料夾中進行，測試結束後會自動清理。

### Q: 可以在生產環境執行測試嗎？
A: 可以，但建議在測試環境執行。測試不會修改現有的 Git 配置或檔案。

### Q: 如何新增自定義測試？
A: 
1. 在 `unit-tests.sh` 中新增 `test_your_function()` 函數
2. 在 `run_tests()` 中調用新函數
3. 使用提供的 `assert_*` 函數進行驗證
